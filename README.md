# app-bullfighting-guide

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态网页形式存在，不依附于任何特定域名或网站。仓库主要用于存放、整理和共享这些页面内容，便于长期保存与访问。

## 目录说明

```
.
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（图片、样式、脚本等）
└── README.md       # 本文件
```

- `pages/` 目录下每个 HTML 文件均为一个独立页面，文件名即为页面标识。
- `assets/` 目录存放页面所需的公共资源，按需组织子目录。

## 页面归档说明

- 所有页面均为纯前端实现，不依赖后端服务。
- 页面之间相互独立，无内部链接依赖。
- 归档时保留原始文件结构与编码格式，不做额外修改。
- 如需新增页面，请将 HTML 文件放置于 `pages/` 目录下，并确保引用的资源路径正确（相对于仓库根目录）。

## 维护说明

- 本仓库仅用于静态页面归档，不提供动态功能或交互服务。
- 页面内容可能随时间调整或更新，更新记录可通过 Git 提交历史查看。
- 如发现页面显示异常，请检查资源文件是否完整，或确认浏览器兼容性。
- 欢迎通过 Issue 提交问题反馈或改进建议。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接使用浏览器打开 `pages/` 目录下的任意 HTML 文件即可查看页面内容。
3. 也可将整个仓库部署至任意静态文件服务器（如 GitHub Pages、Netlify 等）进行在线访问。

本仓库内容仅供学习与参考使用，不涉及商业用途。具体页面可能包含各自独立的版权声明，请以页面内标注为准。
